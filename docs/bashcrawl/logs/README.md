# Logs Directory

Gameplay telemetry, session data, and test screenshots. All runtime files are gitignored — they are generated locally during play or testing and should not be committed.

## Directory Structure

```text
logs/
├── README.md                  # This documentation (tracked)
├── github-builder.log         # Development build log (tracked)
├── sessions/                  # JSONL session telemetry (gitignored)
│   └── 2026-02-21_71695588.jsonl
├── screenshots/               # SVG test screenshots (gitignored)
│   ├── .gitkeep
│   └── 2026-02-21_095855_full_critical_path/
│       ├── 000_initial.svg
│       ├── 001_cd_entrance.svg
│       └── manifest.json
└── feedback/                  # Markdown analytics reports (gitignored)
    └── 2026-02-21_71695588.md
```

## Log Types

### Session Logs (`sessions/*.jsonl`)

Generated by `lib/log.sh`. Each line is a JSON object:

```json
{"ts":"2026-02-16T14:03:22","sid":"71266674","event":"room_enter","room":"cellar","inventory":"amulet,","hp":15}
```

Fields: `ts` (ISO timestamp), `sid` (session ID), `event` (event type), `room`, `inventory`, `hp`, plus any extra key-value pairs.

### Screenshot Captures (`screenshots/<test-name>/`)

Generated by the test framework (`test/fixtures/screenshot_capture.py`) and the agent mode (`ti.agent`). Each test run creates a timestamped subdirectory containing SVG screenshots and a `manifest.json` index:

```json
{
  "generated_at": "2026-02-21T09:30:15",
  "screenshot_dir": "logs/screenshots/2026-02-21_093015_...",
  "total_screenshots": 14,
  "total_size_bytes": 980000,
  "screenshots": [
    {"path": "...", "name": "000_initial.svg", "trigger": "agent_auto"}
  ]
}
```

To browse screenshots locally:

```bash
python3 -m http.server 8899 --directory logs/screenshots/
```

### Feedback Reports (`feedback/*.md`)

Generated by `lib/report.sh`. Markdown summaries of session analytics.

### Runtime Logs (root of `logs/`)

- `bashcrawl.log` — plain-text log from `main.sh` launcher
- `setup.log` — plain-text log from `setup.sh`
- `session-*.log` — legacy plain-text session logs

These are all gitignored and removed during cleanup.

## Gitignore Rules

All session data, screenshots, feedback reports, and runtime logs are excluded from version control via `.gitignore`. Only these files are tracked:

- `logs/README.md`
- `logs/github-builder.log`
- `logs/screenshots/.gitkeep`

## Cleanup

Use the cleanup script to remove all generated log data:

```bash
bash lib/clean_logs.sh              # Remove all generated logs
bash lib/clean_logs.sh --dry        # Preview what would be removed
bash lib/clean_logs.sh --sessions   # Remove only session JSONL files
bash lib/clean_logs.sh --screenshots # Remove only screenshot directories
bash lib/clean_logs.sh --keep 10    # Keep the 10 most recent sessions
```

To reset game state (unlocked rooms, inventory, etc.) use:

```bash
bash lib/reset.sh                   # Reset game state
bash lib/reset.sh --dry             # Preview reset actions
```

## Generating Test Data

```bash
bash lib/test_logging.sh            # Create sample session files
```

```bash
bash lib/analyze.sh                 # Analyze session data
```
