# Logs Directory

Gameplay and playtest telemetry. All runtime files are gitignored — they are generated locally during play or testing and should not be committed.

## Directory Structure

```text
logs/
├── README.md                  # This documentation (tracked)
└── sessions/                  # JSONL session telemetry (gitignored)
    ├── 2026-02-21_71695588.jsonl
    └── blank_slate/           # MCP playtest harness recordings
        └── <session>.jsonl
```

## Session Logs (`sessions/*.jsonl`)

Generated by `lib/log.sh` when game scripts and tests run. Each line is a JSON object:

```json
{"ts":"2026-02-16T14:03:22","sid":"71266674","event":"room_enter","room":"cellar","inventory":"amulet,","hp":15}
```

Fields: `ts` (ISO timestamp), `sid` (session ID), `event` (event type), `room`, `inventory`, `hp`, plus any extra key-value pairs.

## Playtest Recordings (`sessions/blank_slate/*.jsonl`)

Written by the MCP playtest harness (`src/playtest/`) while an external agent plays the game. One JSONL line per event: each command turn plus classified events —

- `discovery` — first productive use of a new command verb / first room
- `struggle` — an error or a no-progress repeat
- `content_gap` — repeated struggles in one spot, or an agent-reported gap

Run the harness and score the recordings:

```bash
PYTHONPATH=src python3 -m playtest.mcp_server     # serve the game over MCP
PYTHONPATH=src python3 -m playtest.scorer --log-dir logs/sessions/blank_slate
```

## Cleanup

Remove generated logs and session data with `make clean-all`. To reset only game state (unlocked rooms, defeat flags, etc.):

```bash
bash lib/reset.sh                   # Reset game state
bash lib/reset.sh --dry             # Preview reset actions
```
