# Script Output Directory

This directory contains output files generated by the documentation analysis and harmonization scripts located in `README/scripts/`.

## Directory Structure

```
output/
├── analysis/          # Documentation structure analysis results
├── recommendations/   # AI-generated recommendations for doc improvements
└── temp/             # Temporary and archived output files
```

## Subdirectories

### `analysis/`

Contains output from documentation structure analysis scripts:

- **`doc_structure_analysis.json`**: Complete analysis of documentation structure, including file counts, duplicates, empty files, and organization recommendations
- Generated by: `scripts/analyze_doc_structure.py`
- Purpose: Identify structural issues, duplicates, and opportunities for reorganization

**Example usage:**
```bash
cd README
python scripts/analyze_doc_structure.py --docs-dir ../docs --output output/analysis/doc_structure_analysis.json
```

### `recommendations/`

Stores AI-generated recommendations for documentation improvements:

- **`recommendations_*.json`**: Structured recommendations with actions (merge, delete, split, enhance)
- Includes priority levels (critical, high, medium, low)
- Contains detailed analysis, quality scores, and improvement suggestions
- Generated by: `scripts/harmonize_docs.py`

**Example usage:**
```bash
cd README
python scripts/harmonize_docs.py export --session-id <session_id> --output output/recommendations/recommendations_<session_id>.json
```

### `temp/`

Archives and temporary files:

- Old output files before reorganization
- Test exports and experimental results
- Files prefixed with `.temp_` or generated during development
- Safe to clean periodically

**Cleanup:**
```bash
# Remove all temporary files
rm -rf README/output/temp/*

# Keep .gitkeep file
touch README/output/temp/.gitkeep
```

## Git Configuration

The following patterns are excluded from version control (see `.gitignore`):

```gitignore
# Script output files
output/temp/
output/analysis/*.json
output/recommendations/*.json
!output/analysis/.gitkeep
!output/recommendations/.gitkeep
```

This ensures:
- ✅ Directory structure is preserved in git
- ✅ Generated files are not committed
- ✅ Local development outputs remain local
- ✅ Clean repository without build artifacts

## Script Integration

All scripts in `README/scripts/` have been updated to use these output directories by default:

| Script | Default Output Path |
|--------|-------------------|
| `analyze_doc_structure.py` | `output/analysis/doc_structure_analysis.json` |
| `harmonize_docs.py` | `output/recommendations/recommendations_<session_id>.json` |
| `harmonize/processor.py` | `output/recommendations/recommendations.json` |

## Workflow

1. **Run Analysis**: Generate structure analysis
   ```bash
   python scripts/analyze_doc_structure.py
   ```

2. **Review Results**: Check `output/analysis/` for findings

3. **Generate Recommendations**: Use harmonization engine
   ```bash
   python scripts/harmonize_docs.py analyze --batch-size 10
   python scripts/harmonize_docs.py export --session-id <id>
   ```

4. **Review Recommendations**: Check `output/recommendations/` for AI suggestions

5. **Apply Changes**: Use processor to implement approved recommendations
   ```bash
   python scripts/harmonize_docs.py apply --session-id <id>
   ```

6. **Archive**: Move processed files to `output/temp/` if needed

## Best Practices

- **Keep Current**: Only keep the most recent analysis and recommendations
- **Archive Old Files**: Move outdated files to `temp/` before generating new ones
- **Clean Regularly**: Periodically clean `temp/` directory
- **Document Custom Outputs**: If adding new output types, update this README
- **Backup Important Results**: Save significant findings outside this directory before cleanup
